## Fix: Template Literal Escaping Bug (Current Session)

### Problem
Bundles were generating JavaScript with `\${...}` instead of `${...}` in template literals, causing:
```
Uncaught SyntaxError: missing ) after argument list
```

### Root Cause
In `esbuild.js`, the `escapeBundleContent()` function was double-escaping template literals:
1. `escapeBundleContent` converted `${` to `\${`
2. `JSON.stringify` then escaped the backslash to `\\${`
3. When parsed, this became `\${` which is invalid inside template literals

### Fix
Removed the unnecessary `escapeBundleContent()` function from `esbuild.js`. `JSON.stringify()` already handles string escaping correctly - no additional escaping is needed.

### Files Modified
- `esbuild.js`: Removed `escapeBundleContent()` function and its usage

### Verification
- Rebuilt bundles: `node esbuild.js`
- Confirmed template literals are now correct: `id:\`v5-${Date.now()}-...\``
- No more `\\${` double-escaping in inline bundle files
